## What Feast does

Feast solves three core problems for ML platform teams:

**Training-serving consistency** — Feast manages an offline store for historical feature retrieval and an online store for low-latency inference, ensuring the same feature logic powers both training and serving.

**Point-in-time correctness** — When generating training datasets, Feast joins features using point-in-time logic so future values never leak into your model's training data.

**Infrastructure decoupling** — Feast abstracts feature storage and retrieval so your models remain portable across data infrastructure changes — from batch to real-time, from one cloud to another.

## Get started

<Steps>
  <Step title="Install Feast">
    ```bash
    pip install feast
    ```
  </Step>
  <Step title="Create a feature repository">
    ```bash
    feast init my_feature_repo
    cd my_feature_repo/feature_repo
    ```
  </Step>
  <Step title="Apply your feature definitions">
    ```bash
    feast apply
    ```
  </Step>
  <Step title="Retrieve features">
    ```python
    from feast import FeatureStore

    store = FeatureStore(repo_path=".")
    features = store.get_online_features(
        features=["driver_hourly_stats:conv_rate"],
        entity_rows=[{"driver_id": 1001}],
    ).to_dict()
    ```
  </Step>
</Steps>
