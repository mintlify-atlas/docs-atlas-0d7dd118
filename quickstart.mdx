This guide walks you through a complete Feast workflow using a local setup: Parquet files as the offline store and SQLite as the online store. By the end you will have generated a training dataset, materialized features to the online store, and retrieved them for real-time inference.

<Note>
  For production deployments on AWS, GCP, or Snowflake, see the [cloud providers guide](/deployment/cloud-providers) after completing this tutorial.
</Note>

## What you will build

A feature store for a ride-sharing driver satisfaction model. The store holds hourly driver statistics — conversion rate, acceptance rate, and average daily trips — that power both model training and real-time inference.

<Steps>

<Step title="Install Feast">

Create a virtual environment, then install the Feast SDK and CLI:

```bash
python -m venv venv/
source venv/bin/activate
pip install feast
```

Verify the installation:

```bash
feast version
```

</Step>

<Step title="Create a feature repository">

Bootstrap a new feature repository with `feast init`. This generates a working example project with sample data, feature definitions, and a configuration file.

```bash
feast init my_project
cd my_project/feature_repo
```

The generated repository contains:

```
my_project/
└── feature_repo/
    ├── data/
    │   └── driver_stats.parquet   # Sample driver statistics data
    ├── feature_definitions.py     # Feature views, entities, and services
    ├── feature_store.yaml         # Store configuration
    └── test_workflow.py           # End-to-end workflow script
```

<CodeGroup>

```yaml feature_store.yaml
project: my_project
# File-based registry stores feature definitions locally.
# For production, switch to a SQL-backed registry.
registry: data/registry.db
# The provider sets default offline and online stores.
# local = file/Dask offline store + SQLite online store
provider: local
online_store:
  type: sqlite
  path: data/online_store.db
entity_key_serialization_version: 3
```

```python feature_definitions.py
from datetime import timedelta

import pandas as pd

from feast import (
    Entity,
    FeatureService,
    FeatureView,
    Field,
    FileSource,
    Project,
    PushSource,
    RequestSource,
)
from feast.on_demand_feature_view import on_demand_feature_view
from feast.types import Float32, Float64, Int64

# Define a project for the feature repo
project = Project(name="my_project", description="A project for driver statistics")

# An entity is the primary key used to look up features — here, a driver.
driver = Entity(name="driver", join_keys=["driver_id"])

# Read raw feature data from a local Parquet file.
driver_stats_source = FileSource(
    name="driver_hourly_stats_source",
    path="data/driver_stats.parquet",
    timestamp_field="event_timestamp",
    created_timestamp_column="created",
)

# A FeatureView defines the schema and source for a set of features.
driver_stats_fv = FeatureView(
    name="driver_hourly_stats",
    entities=[driver],
    ttl=timedelta(days=1),
    schema=[
        Field(name="conv_rate", dtype=Float32),
        Field(name="acc_rate", dtype=Float32),
        Field(name="avg_daily_trips", dtype=Int64, description="Average daily trips"),
    ],
    online=True,
    source=driver_stats_source,
    tags={"team": "driver_performance"},
)

# A RequestSource holds features available only at request time.
input_request = RequestSource(
    name="vals_to_add",
    schema=[
        Field(name="val_to_add", dtype=Int64),
        Field(name="val_to_add_2", dtype=Int64),
    ],
)

# An on-demand feature view computes derived features at retrieval time.
@on_demand_feature_view(
    sources=[driver_stats_fv, input_request],
    schema=[
        Field(name="conv_rate_plus_val1", dtype=Float64),
        Field(name="conv_rate_plus_val2", dtype=Float64),
    ],
)
def transformed_conv_rate(inputs: pd.DataFrame) -> pd.DataFrame:
    df = pd.DataFrame()
    df["conv_rate_plus_val1"] = inputs["conv_rate"] + inputs["val_to_add"]
    df["conv_rate_plus_val2"] = inputs["conv_rate"] + inputs["val_to_add_2"]
    return df

# A FeatureService groups features into a versioned, named set for a model.
driver_activity_v1 = FeatureService(
    name="driver_activity_v1",
    features=[
        driver_stats_fv[["conv_rate"]],  # Sub-selects a single feature
        transformed_conv_rate,
    ],
)

# A PushSource allows streaming data to be written directly to the online store.
driver_stats_push_source = PushSource(
    name="driver_stats_push_source",
    batch_source=driver_stats_source,
)
```

</CodeGroup>

The `provider` field in `feature_store.yaml` determines the default offline and online stores. Valid values are `local`, `gcp`, and `aws`. Third-party community plugins add support for Spark, Azure, Hive, Trino, PostgreSQL, and more.

</Step>

<Step title="Generate a training dataset">

Use `store.get_historical_features()` to build a training dataset. You supply an entity dataframe that maps entity values to timestamps. Feast performs a point-in-time join: for each row it retrieves the feature values that were valid at or before the given timestamp, ensuring no future data leaks into your training set.

```python
from datetime import datetime

import pandas as pd
from feast import FeatureStore

entity_df = pd.DataFrame.from_dict(
    {
        # Entity join key and values
        "driver_id": [1001, 1002, 1003],
        # event_timestamp is reserved — Feast uses it as the upper bound
        # for the point-in-time join. No feature value observed after this
        # timestamp will appear in the resulting row.
        "event_timestamp": [
            datetime(2021, 4, 12, 10, 59, 42),
            datetime(2021, 4, 12, 8, 12, 10),
            datetime(2021, 4, 12, 16, 40, 26),
        ],
        # Labels are passed through unchanged — Feast does not process them
        "label_driver_reported_satisfaction": [1, 5, 3],
        # Request-time values used by the on-demand feature view
        "val_to_add": [1, 2, 3],
        "val_to_add_2": [10, 20, 30],
    }
)

store = FeatureStore(repo_path=".")

training_df = store.get_historical_features(
    entity_df=entity_df,
    features=[
        "driver_hourly_stats:conv_rate",
        "driver_hourly_stats:acc_rate",
        "driver_hourly_stats:avg_daily_trips",
        "transformed_conv_rate:conv_rate_plus_val1",
        "transformed_conv_rate:conv_rate_plus_val2",
    ],
).to_df()

print(training_df.head())
```

<Tip>
  For batch scoring, set `entity_df["event_timestamp"] = pd.to_datetime("now", utc=True)` before calling `get_historical_features()`. This retrieves the most recent feature values for each entity.
</Tip>

</Step>

<Step title="Materialize features to the online store">

Materialization copies the latest feature values from the offline store into the online store so they can be retrieved with low latency at inference time.

`feast materialize-incremental` transfers all new data since the last materialization run up to the given timestamp. The `ttl` set on each `FeatureView` determines the lookback window.

<Tabs>
  <Tab title="CLI (incremental)">
    ```bash
    CURRENT_TIME=$(date -u +"%Y-%m-%dT%H:%M:%S")
    feast materialize-incremental $CURRENT_TIME
    ```
  </Tab>
  <Tab title="CLI (all data)">
    ```bash
    # Materialize all data ignoring event timestamps
    feast materialize --disable-event-timestamp
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    from datetime import datetime
    from feast import FeatureStore

    store = FeatureStore(repo_path=".")
    store.materialize_incremental(end_date=datetime.utcnow())
    ```
  </Tab>
</Tabs>

<Note>
  The online store keeps only the **latest** feature value per entity key. No historical values are stored there — use `get_historical_features()` for time-travel queries.
</Note>

</Step>

<Step title="Retrieve online features for inference">

At inference time, use `store.get_online_features()` to fetch the latest materialized values with low latency. Results are returned as an `OnlineResponse` object that can be converted to a dict or DataFrame.

<CodeGroup>

```python Using feature references
from pprint import pprint
from feast import FeatureStore

store = FeatureStore(repo_path=".")

feature_vector = store.get_online_features(
    features=[
        "driver_hourly_stats:conv_rate",
        "driver_hourly_stats:acc_rate",
        "driver_hourly_stats:avg_daily_trips",
    ],
    entity_rows=[
        # Each dict maps the entity join key to a value
        {"driver_id": 1004},
        {"driver_id": 1005},
    ],
).to_dict()

pprint(feature_vector)
```

```python Using a feature service
from pprint import pprint
from feast import FeatureStore

store = FeatureStore(repo_path=".")

# A FeatureService decouples model requirements from feature view definitions.
feature_service = store.get_feature_service("driver_activity_v1")

feature_vector = store.get_online_features(
    features=feature_service,
    entity_rows=[
        {"driver_id": 1004},
        {"driver_id": 1005},
    ],
).to_dict()

pprint(feature_vector)
```
