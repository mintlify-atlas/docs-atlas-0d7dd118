---
title: "Feature View Types"
description: "Reference for all Feast feature view types: FeatureView, OnDemandFeatureView, StreamFeatureView, BatchFeatureView, and FeatureService."
---

Feature views are the core abstraction in Feast for grouping and managing features. Each type is suited to a different data pipeline pattern. They all share the concept of a `schema` (a list of `Field` objects) and an association with one or more `Entity` objects.

---

## Field and Supported Types

Before defining feature views, declare the output schema using `Field` objects.

```python
from feast.field import Field
from feast.types import (
    String, Int32, Int64, Float32, Float64, Bool,
    Bytes, UnixTimestamp, Array, Invalid,
)
```

<Accordion title="Field">

```python
Field(
    name: str,
    dtype: FeatureSpecV2,          # e.g. Float32, Int64, String
    description: str = "",
    tags: Optional[Dict[str, str]] = None,
)
```

Describes a single column in a feature view schema.

<ParamField path="name" type="str" required>
  Column name as it appears in the data source and will appear in retrieval output.
</ParamField>

<ParamField path="dtype" type="FeatureSpecV2" required>
  The Feast type for this column. Import from `feast.types`.

  <Expandable title="Supported types">
    | Type | Python import | Notes |
    |---|---|---|
    | `Int32` | `from feast.types import Int32` | 32-bit integer |
    | `Int64` | `from feast.types import Int64` | 64-bit integer |
    | `Float32` | `from feast.types import Float32` | 32-bit float |
    | `Float64` | `from feast.types import Float64` | 64-bit float / double |
    | `String` | `from feast.types import String` | UTF-8 string |
    | `Bool` | `from feast.types import Bool` | Boolean |
    | `Bytes` | `from feast.types import Bytes` | Raw bytes |
    | `UnixTimestamp` | `from feast.types import UnixTimestamp` | Epoch seconds |
    | `Array(inner)` | `from feast.types import Array` | Fixed-type list, e.g. `Array(Float32)` |
    | `Invalid` | `from feast.types import Invalid` | Unknown/unset type |
  </Expandable>
</ParamField>

<ParamField path="description" type="str">
  Human-readable description displayed in the Feast UI.
</ParamField>

<ParamField path="tags" type="Optional[Dict[str, str]]">
  Arbitrary key-value metadata attached to this field.
</ParamField>

</Accordion>

## FeatureView

The standard batch feature view. Reads from an offline data source (file, BigQuery, Snowflake, etc.) and optionally materializes to an online store.

```python
from feast import FeatureView
```

<Accordion title="Constructor">

```python
FeatureView(
    *,
    name: str,
    source: Optional[Union[DataSource, FeatureView, List[FeatureView]]] = None,
    sink_source: Optional[DataSource] = None,
    schema: Optional[List[Field]] = None,
    entities: Optional[List[Entity]] = None,
    ttl: Optional[timedelta] = timedelta(days=0),
    online: bool = True,
    offline: bool = False,
    description: str = "",
    tags: Optional[Dict[str, str]] = None,
    owner: str = "",
    mode: Optional[Union[TransformationMode, str]] = None,
    enable_validation: bool = False,
    version: str = "latest",
)
```

<ParamField path="name" type="str" required>
  Globally unique name for this feature view within the project.
</ParamField>

<ParamField path="source" type="Optional[Union[DataSource, FeatureView, List[FeatureView]]]">
  The data source. May be a batch `DataSource` (e.g. `FileSource`, `BigQuerySource`), a streaming `PushSource`, a `KafkaSource`, a `KinesisSource`, another `FeatureView`, or a list of `FeatureView` objects. When a stream source (`PushSource`, `KafkaSource`, `KinesisSource`) is provided, the underlying `batch_source` is automatically extracted.
</ParamField>

<ParamField path="schema" type="Optional[List[Field]]">
  The list of output feature columns. If omitted, Feast infers columns from the underlying data source schema at `feast apply` time.
</ParamField>

<ParamField path="entities" type="Optional[List[Entity]]">
  Entities that join this feature view to other data. If omitted, the view is treated as entityless (a global feature view with a dummy entity).
</ParamField>

<ParamField path="ttl" type="Optional[timedelta]">
  Time-to-live for features in the online store. A value of `timedelta(days=0)` means features never expire. Shorter TTLs limit the offline scan window during historical retrieval.
</ParamField>

<ParamField path="online" type="bool">
  Enable online serving and materialization. Default is `True`.
</ParamField>

<ParamField path="offline" type="bool">
  Enable writing to the offline store. Default is `False`.
</ParamField>

<ParamField path="description" type="str">
  Human-readable description shown in the Feast UI.
</ParamField>

<ParamField path="tags" type="Optional[Dict[str, str]]">
  Arbitrary key-value metadata. Supports filtering via `list_feature_views(tags={...})`.
</ParamField>

<ParamField path="owner" type="str">
  Owner contact, typically an email address.
</ParamField>

<ParamField path="mode" type="Optional[Union[TransformationMode, str]]">
  Transformation mode applied when this view is used as a source for derived views. Options from `TransformationMode`: `PYTHON`, `PANDAS`, `RAY`, `SQL`, `SPARK`, `SUBSTRAIT`.
</ParamField>

<ParamField path="enable_validation" type="bool">
  When `True`, validates incoming data against the declared schema during materialization. Default is `False`.
</ParamField>

<ParamField path="version" type="str">
  Version string for definition management (e.g., `"v2"`). Only one version is active per project per feature view name. Default is `"latest"`.
</ParamField>

</Accordion>

```python
from feast import FeatureView, Entity, FileSource
from feast.field import Field
from feast.types import Float32, Int64, String
from datetime import timedelta

driver = Entity(name="driver_id", description="Driver identifier")

driver_stats_source = FileSource(
    path="data/driver_stats.parquet",
    timestamp_field="event_timestamp",
    created_timestamp_column="created",
)

driver_stats_fv = FeatureView(
    name="driver_hourly_stats",
    entities=[driver],
    ttl=timedelta(days=1),
    schema=[
        Field(name="conv_rate",       dtype=Float32),
        Field(name="acc_rate",        dtype=Float32),
        Field(name="avg_daily_trips", dtype=Int64),
    ],
    source=driver_stats_source,
    tags={"team": "driver_performance"},
    description="Hourly aggregated driver stats",
)
```

## OnDemandFeatureView

Computes features at retrieval time by applying a user-defined function (UDF) over existing feature views and/or request-time data. No pre-materialization is required by default.

```python
from feast import OnDemandFeatureView
from feast.on_demand_feature_view import on_demand_feature_view
```

<Accordion title="@on_demand_feature_view decorator">

```python
@on_demand_feature_view(
    *,
    name: Optional[str] = None,
    entities: Optional[List[Entity]] = None,
    schema: list[Field],
    sources: list[Union[FeatureView, RequestSource, FeatureViewProjection]],
    mode: str = "pandas",
    description: str = "",
    tags: Optional[dict[str, str]] = None,
    owner: str = "",
    write_to_online_store: bool = False,
    singleton: bool = False,
    track_metrics: bool = False,
    explode: bool = False,
    version: str = "latest",
)
```

<ParamField path="schema" type="list[Field]" required>
  The output columns produced by the transformation function.
</ParamField>

<ParamField path="sources" type="list[Union[FeatureView, RequestSource, FeatureViewProjection]]" required>
  Input sources for the UDF. The UDF receives each source as a pandas DataFrame (in `"pandas"` mode) or dict (in `"python"` mode), keyed by the source's name.
</ParamField>

<ParamField path="mode" type="str">
  Execution mode. `"pandas"` (default) â€” UDF receives and returns `pd.DataFrame`. `"python"` â€” UDF receives and returns a plain Python dict.
</ParamField>

<ParamField path="write_to_online_store" type="bool">
  When `True`, the transformed output can be materialized to the online store (precomputed). Requires at least one entity. Default is `False`.
</ParamField>

<ParamField path="singleton" type="bool">
  When `True` and `mode="python"`, the UDF is called once per row rather than once per batch. Default is `False`.
</ParamField>

<ParamField path="track_metrics" type="bool">
  Emit Prometheus duration metrics (`feast_feature_server_transformation_duration_seconds`) for this view. Default is `False`.
</ParamField>

</Accordion>

<CodeGroup>

```python Pandas mode
import pandas as pd
from feast import FeatureView
from feast.field import Field
from feast.types import Float64
from feast.on_demand_feature_view import on_demand_feature_view

# Assumes driver_hourly_stats FeatureView is already defined
@on_demand_feature_view(
    sources=[driver_hourly_stats],
    schema=[Field(name="conv_rate_adjusted", dtype=Float64)],
    mode="pandas",
)
def driver_stats_transformed(inputs: pd.DataFrame) -> pd.DataFrame:
    df = pd.DataFrame()
    df["conv_rate_adjusted"] = inputs["conv_rate"] * 1.1
    return df
```

```python Python mode (per-row)
from feast.field import Field
from feast.types import Float64
from feast.on_demand_feature_view import on_demand_feature_view

@on_demand_feature_view(
    sources=[driver_hourly_stats],
    schema=[Field(name="conv_rate_adjusted", dtype=Float64)],
    mode="python",
    singleton=True,
)
def driver_stats_transformed(row: dict) -> dict:
    return {"conv_rate_adjusted": row["conv_rate"] * 1.1}
```

```python With RequestSource
from feast.data_source import RequestSource
from feast.field import Field
from feast.types import Float64, Int32
from feast.on_demand_feature_view import on_demand_feature_view

request_source = RequestSource(
    name="trip_request",
    schema=[Field(name="trip_distance", dtype=Float64)],
)

@on_demand_feature_view(
    sources=[driver_hourly_stats, request_source],
    schema=[Field(name="adjusted_rate", dtype=Float64)],
    mode="pandas",
)
def adjusted_driver_rate(
    driver_hourly_stats: pd.DataFrame,
    trip_request: pd.DataFrame,
) -> pd.DataFrame:
    df = pd.DataFrame()
    df["adjusted_rate"] = (
        driver_hourly_stats["conv_rate"] / trip_request["trip_distance"]
    )
    return df
```

</CodeGroup>

## StreamFeatureView

Defines features that have both a **stream** source (Kafka or PushSource) and a **batch** source. Used for real-time feature computation with optional UDF transformations and windowed aggregations.

```python
from feast import StreamFeatureView
from feast.stream_feature_view import stream_feature_view
```

<Warning>
Stream feature views are experimental features in alpha development. The API is stable but functionality may change.
</Warning>

<Accordion title="@stream_feature_view decorator">

```python
@stream_feature_view(
    *,
    entities: Optional[Union[List[Entity], List[str]]] = None,
    ttl: Optional[timedelta] = None,
    tags: Optional[Dict[str, str]] = None,
    online: Optional[bool] = True,
    description: Optional[str] = "",
    owner: Optional[str] = "",
    schema: Optional[List[Field]] = None,
    source: Optional[DataSource] = None,
    aggregations: Optional[List[Aggregation]] = None,
    mode: Optional[str] = "spark",
    timestamp_field: Optional[str] = "",
    enable_validation: bool = False,
    version: str = "latest",
)
```

<ParamField path="source" type="Optional[DataSource]">
  The stream data source. Must be a `KafkaSource` or `PushSource` (or a `CUSTOM_SOURCE`). A batch source is automatically derived from the stream source's `batch_source` attribute.
</ParamField>

<ParamField path="schema" type="Optional[List[Field]]">
  Output schema. Schema inference is not supported for `StreamFeatureView`; you must declare the schema explicitly.
</ParamField>

<ParamField path="entities" type="Optional[Union[List[Entity], List[str]]]">
  Entities associated with this feature view.
</ParamField>

<ParamField path="ttl" type="Optional[timedelta]">
  Time-to-live for features in the online store.
</ParamField>

<ParamField path="aggregations" type="Optional[List[Aggregation]]">
  Windowed aggregations to apply to the stream (e.g., sum, count). Requires `timestamp_field` to be set.
</ParamField>

<ParamField path="timestamp_field" type="Optional[str]">
  The column to use as the event timestamp for windowed aggregations.
</ParamField>

<ParamField path="mode" type="Optional[str]">
  Transformation mode. Default is `"spark"`. Options: `"spark"`, `"python"`, `"pandas"`, `"spark_sql"`.
</ParamField>

</Accordion>

```python
import pandas as pd
from datetime import timedelta
from feast import Entity, Field
from feast.types import Float32, Int64
from feast.stream_feature_view import stream_feature_view
from feast.data_source import KafkaSource, KafkaOptions

driver = Entity(name="driver_id")

kafka_source = KafkaSource(
    name="driver_stats_stream",
    kafka_bootstrap_servers="localhost:9092",
    topic="driver-stats",
    batch_source=driver_stats_source,   # fallback batch source
    timestamp_field="event_timestamp",
    message_format=JsonFormat(schema_json="..."),
)

@stream_feature_view(
    entities=[driver],
    ttl=timedelta(hours=24),
    mode="spark",
    schema=[
        Field(name="conv_rate",       dtype=Float32),
        Field(name="acc_rate",        dtype=Float32),
        Field(name="avg_daily_trips", dtype=Int64),
    ],
    source=kafka_source,
)
def driver_stats_stream(df: pd.DataFrame) -> pd.DataFrame:
    return df
```

## BatchFeatureView

An explicit batch-only feature view. Semantically marks that a feature view exclusively reads from batch sources. Supports UDF transformations and aggregations.

```python
from feast import BatchFeatureView
from feast.batch_feature_view import batch_feature_view
```

<Warning>
Batch feature views are experimental features in alpha development.
</Warning>

<Accordion title="Constructor and decorator">

```python
BatchFeatureView(
    *,
    name: str,
    mode: Union[TransformationMode, str] = TransformationMode.PYTHON,
    source: Optional[Union[DataSource, BatchFeatureView, List[BatchFeatureView]]] = None,
    sink_source: Optional[DataSource] = None,
    entities: Optional[List[Entity]] = None,
    ttl: Optional[timedelta] = None,
    tags: Optional[Dict[str, str]] = None,
    online: bool = False,
    offline: bool = False,
    description: str = "",
    owner: str = "",
    schema: Optional[List[Field]] = None,
    udf: Optional[Callable[[Any], Any]] = None,
    udf_string: Optional[str] = "",
    feature_transformation: Optional[Transformation] = None,
    batch_engine: Optional[Dict[str, Any]] = None,
    aggregations: Optional[List[Aggregation]] = None,
    enable_validation: bool = False,
    version: str = "latest",
)
```

Supported `source` types: `BigQuerySource`, `FileSource`, `RedshiftSource`, `SnowflakeSource`, `SparkSource`, `TrinoSource`, `AthenaSource`.

<ParamField path="source" type="Optional[Union[DataSource, BatchFeatureView, List[BatchFeatureView]]]">
  A batch data source. Must be one of the supported batch source types.
</ParamField>

<ParamField path="mode" type="Union[TransformationMode, str]">
  Transformation mode. Options: `PYTHON` (default), `PANDAS`, `SQL`, `RAY`.
</ParamField>

<ParamField path="batch_engine" type="Optional[Dict[str, Any]]">
  Override the repo-level batch engine configuration for this specific view.
</ParamField>

<ParamField path="udf" type="Optional[Callable[[Any], Any]]">
  User-defined transformation function. All non-built-in imports must be inside the function body for correct serialization.
</ParamField>

</Accordion>

```python
import pandas as pd
from feast.batch_feature_view import batch_feature_view
from feast.field import Field
from feast.types import Float32, Int64
from datetime import timedelta

@batch_feature_view(
    entities=[driver],
    ttl=timedelta(days=7),
    mode="pandas",
    schema=[
        Field(name="lifetime_conv_rate", dtype=Float32),
        Field(name="lifetime_trips",     dtype=Int64),
    ],
    source=driver_stats_source,
    online=True,
)
def driver_lifetime_stats(df: pd.DataFrame) -> pd.DataFrame:
    import pandas as pd
    result = df.groupby("driver_id").agg(
        lifetime_conv_rate=("conv_rate", "mean"),
        lifetime_trips=("avg_daily_trips", "sum"),
    ).reset_index()
    return result
```

## FeatureService

Groups features from multiple feature views for model versioning and retrieval. A `FeatureService` can be passed directly to `get_historical_features` or `get_online_features`.

```python
from feast import FeatureService
```

<Accordion title="Constructor">

```python
FeatureService(
    *,
    name: str,
    features: List[Union[FeatureView, OnDemandFeatureView]],
    tags: Optional[Dict[str, str]] = None,
    description: str = "",
    owner: str = "",
    logging_config: Optional[LoggingConfig] = None,
)
```

<ParamField path="name" type="str" required>
  Unique name of the feature service within the project.
</ParamField>

<ParamField path="features" type="List[Union[FeatureView, OnDemandFeatureView]]" required>
  Feature views (or projections of feature views) to include. Use Python slice notation to select a subset of features from a view: `fv[["col_a", "col_b"]]`.
</ParamField>

<ParamField path="tags" type="Optional[Dict[str, str]]">
  Arbitrary key-value metadata.
</ParamField>

<ParamField path="description" type="str">
  Human-readable description.
</ParamField>

<ParamField path="owner" type="str">
  Owner contact.
</ParamField>

<ParamField path="logging_config" type="Optional[LoggingConfig]">
  Configuration for feature logging. When set, online serving requests can be logged to the offline store for training data generation.
</ParamField>

</Accordion>

```python
from feast import FeatureService

# Include all features from two views
driver_activity_v1 = FeatureService(
    name="driver_activity_v1",
    features=[driver_hourly_stats, driver_lifetime_stats],
    description="All driver features for model v1",
    tags={"model_version": "v1"},
)

# Include a subset of features from one view
driver_activity_v2 = FeatureService(
    name="driver_activity_v2",
    features=[
        driver_hourly_stats[["conv_rate", "acc_rate"]],
        driver_stats_transformed,  # OnDemandFeatureView
    ],
    description="Curated features for model v2",
)

store.apply([driver_activity_v1, driver_activity_v2])

# Use in retrieval
response = store.get_online_features(
    features=driver_activity_v1,
    entity_rows=[{"driver_id": 1001}],
)
```
